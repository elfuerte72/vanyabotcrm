# Telegram Bot — Архитектура и принцип работы

## Что это за бот?

AI-консультант по питанию. Пользователь пишет боту, бот через ИИ (Gemini 3 Flash) собирает данные (пол, возраст, вес, рост, уровень активности, цель), рассчитывает КБЖУ и генерирует персональный план питания. После этого запускается **воронка продаж** — 5 дней бот присылает сообщения с предложением купить полную программу.

Поддерживаемые языки: русский, английский, арабский.

## Как работает бот — пошагово

### 1. Запуск (`main.py`)

При запуске параллельно стартуют **три процесса**:

| Процесс | Что делает |
|---------|-----------|
| **Polling** | Бот слушает сообщения из Telegram (aiogram) |
| **Webhook-сервер** | aiohttp на порту 8080 — принимает уведомления об оплате от Ziina |
| **Планировщик** | APScheduler — каждый день в 23:00 UTC рассылает воронку |

### 2. Пользователь пишет `/start`

**Файл:** `handlers/start.py`

1. Определяется язык пользователя (по настройкам Telegram)
2. Отправляется приветственное сообщение на нужном языке (RU/EN/AR)

### 3. Пользователь пишет текст или голосовое

**Файл:** `handlers/message.py` — это **ядро бота**.

Каждое сообщение проходит через цепочку:

```
Сообщение → Middlewares → Handler
```

#### Middlewares (выполняются по очереди):

1. **`logging.py`** — записывает в лог: chat_id, тип контента, длину текста
2. **`user_data.py`** — достаёт юзера из БД по Telegram ID и передаёт в хендлер как `db_user` (или `None` если новый)

#### Handler (`message.py`):

1. Если юзер уже получил план питания (`get_food=TRUE`) → отказ: «вы уже получили меню»
2. Если голосовое сообщение → транскрибируется через AI (OpenRouter Whisper)
3. Текст отправляется в **АГЕНТ ГЛАВНЫЙ** (`ai_agent.py`)
4. Агент ведёт диалог: спрашивает пол, возраст, вес, рост, активность, цель, аллергии
5. Когда все данные собраны — агент возвращает JSON с `is_finished: true`
6. Далее:
   - Рассчитываются КБЖУ через `calculator.py` (формула Миффлина-Сан Жеора)
   - Данные сохраняются в БД
   - Вызывается **АГЕНТ ЕДА** (`ai_food.py`) — генерирует план питания в JSON
   - План валидируется: нет исключённых продуктов, калории ±10% от цели
   - Форматируется в HTML (`formatter.py`) — с эмодзи, макросами, итогами
   - Отправляется пользователю
   - Ставится `get_food=TRUE`, `funnel_stage=0` — **воронка стартует**

### 4. Воронка продаж

**Папка:** `funnel/`

Каждый день в **23:00 UTC** планировщик запускает рассылку:

| День (stage) | Содержание |
|-------------|-----------|
| 0 | «Разбуди своё тело» — бесплатная тренировка |
| 1 | Предложение купить после просмотра видео |
| 2 | Социальное доказательство (отзывы) |
| 3 | Проблема → решение |
| 4 | Обоснование цены |
| 5 | Мягкий дедлайн |

**Механика рассылки** (`sender.py`):
- `get_funnel_targets()` — выбирает всех, кто: `is_buyer=FALSE`, `get_food=TRUE`, `funnel_stage` от 0 до 4
- Сообщения отправляются **пачками по 25** с паузой 1 секунда (защита от блокировки Telegram)
- После отправки `funnel_stage` увеличивается на 1
- Покупатели автоматически исключаются из рассылки

**Важный нюанс:** stage увеличивается планировщиком *после* отправки, поэтому когда пользователь нажимает кнопку из сообщения stage N, он уже находится на stage N+1.

### 5. Нажатие кнопок (callbacks)

**Файл:** `handlers/callbacks.py`

| Callback | Действие |
|----------|---------|
| `buy_now` | Помечает как покупателя, отправляет ссылку на оплату (Tribute для RU, Ziina для EN/AR) |
| `video_workout` | Отправляет видео тренировки, через 5 минут — follow-up с кнопкой «купить» |
| `show_info` | Скачивает и отправляет видео «что входит в программу» с Google Drive |
| `check_suitability` | Скачивает и отправляет видео «подойдёт ли мне» |
| `show_results` | Отправляет случайное фото результата |
| `remind_later` | Отправляет сообщение «хорошо, напомню позже» |
| `none` | Отправляет вежливый отказ |

### 6. Оплата (Ziina webhook)

**Файл:** `handlers/payment.py`

1. Ziina отправляет POST-запрос на `/webhook/ziina`
2. Проверяется подпись HMAC-SHA256 (обязательно — без секрета вебхук возвращает 503)
3. Извлекается `chat_id` из метаданных
4. Если статус `completed` → юзер помечается как покупатель (`mark_as_buyer`)
5. Бот отправляет подтверждение оплаты в чат

## Сервисы — подробное описание

### АГЕНТ ГЛАВНЫЙ (`services/ai_agent.py`)

Ведёт диалог с пользователем через LLM (Gemini 3 Flash via OpenRouter).

- Загружает последние 20 сообщений из `chat_histories`
- Формирует массив сообщений: системный промпт + история + текущее сообщение
- Сохраняет сообщение пользователя в БД *до* вызова API (чтобы не потерять при ошибке)
- Вызывает OpenRouter API
- Сохраняет ответ AI в БД
- Системный промпт задаёт правила: отвечать на языке юзера, задавать 1-2 вопроса за раз, валидировать нереалистичные данные, не отвлекаться на off-topic, итоговый JSON только после подтверждения

### АГЕНТ ЕДА (`services/ai_food.py`)

Генерирует план питания на основе рассчитанных макросов.

- Принимает целевые макросы (калории, белки, жиры, углеводы) и ограничения (аллергии, исключённые продукты)
- Отправляет промпт на языке пользователя
- Требует JSON-ответ (`response_format: json_object`)
- Повторяет до 2 раз при пустом ответе
- Парсит JSON из ответа (обрабатывает и code-блоки, и сырой JSON)
- Возвращает dict с массивом `meals`

### Калькулятор КБЖУ (`services/calculator.py`)

Формула Миффлина-Сан Жеора:

```
Мужчины: BMR = 10 × вес + 6.25 × рост - 5 × возраст + 5
Женщины: BMR = 10 × вес + 6.25 × рост - 5 × возраст - 161
```

Затем:
- BMR × коэффициент активности (от 1.2 «сидячий» до 1.9 «экстремальный»)
- Корректировка по цели: похудение -15%, набор мышц +10%, поддержание 0%
- Жиры: 1 г/кг массы тела
- Белки: 1.3-1.5 г/кг (больше для похудения/набора мышц)
- Остаток калорий → углеводы

### Форматирование (`services/formatter.py`)

- `parse_agent_output()` — извлекает JSON из ответа агента, определяет `is_finished`, конвертирует markdown → HTML
- `markdown_to_telegram_html()` — `**жирный**`, `*курсив*`, списки, санитизация
- `format_meal_plan_html()` — рендерит план питания с эмодзи, калориями, итогами по дню
- `validate_meal_plan()` — проверяет: нет исключённых продуктов, калории в пределах ±10%

### Определение языка (`services/language.py`)

Простая эвристика по Unicode-диапазонам:
- Арабские символы (U+0600–U+077F) → `ar`
- Кириллица (U+0400–U+04FF) → `ru`
- Иначе → `en`

Приоритет: арабский > русский > английский.

### Медиа (`services/media.py`)

Скачивает файлы с Google Drive по ID из `config/media.yaml` и отправляет в Telegram как `BufferedInputFile`. Функции: `send_info_video()`, `send_suitability_video()`, `send_random_result_photo()`.

## База данных

### Подключение (`db/pool.py`)

- Ленивый singleton пул asyncpg (2-10 соединений)
- SSL по умолчанию включён; `sslmode=disable` в URL отключает для локальной разработки
- Потокобезопасная инициализация через `asyncio.Lock`

### SQL-запросы (`db/queries.py`)

| Функция | Описание |
|---------|---------|
| `get_user(chat_id)` | Получить профиль пользователя |
| `save_user_data(...)` | Upsert: создать или обновить юзера с макросами |
| `mark_as_buyer(chat_id)` | Пометить как покупателя (`is_buyer=TRUE`) |
| `set_food_received(chat_id)` | Установить `get_food=TRUE`, `funnel_stage=0`, запустить воронку |
| `get_funnel_targets()` | Все не-покупатели с `get_food=TRUE` и stage 0-4 |
| `update_funnel_stage(chat_id)` | Увеличить stage на 1, обновить timestamp |
| `advance_funnel_if_at_stage(chat_id, stage)` | Условное увеличение (защита от гонки) |
| `get_chat_history(session_id, limit)` | История чата (JSONB, DESC → reverse для LLM) |
| `save_chat_message(session_id, role, content)` | Сохранить сообщение в историю |
| `get_user_language(chat_id)` | Получить язык пользователя |

### Ключевые таблицы

**`users_nutrition`** — профили пользователей:
- PK: `chat_id` (bigint, Telegram ID)
- Данные: пол, возраст, вес, рост, активность, цель
- КБЖУ: калории, белки, жиры, углеводы
- Воронка: `funnel_stage` (0-6), `is_buyer`, `get_food`
- Язык, платёжные данные (Ziina)

**`chat_histories`** — история диалога:
- `session_id` = chat_id как строка
- `message` — JSONB: `{ type: "human"|"ai", content: "..." }`

**`user_events`** — события (нажатия кнопок, действия воронки)

| Переменная | Описание |
|-----------|---------|

### Lazy proxy pattern (`config/settings.py`)

Настройки используют паттерн «ленивый прокси» — объекты `settings` и `media_config` не создаются при импорте модуля. Это позволяет тестам работать без `.env` файла, устанавливая переменные окружения в `conftest.py`.

## Деплой

Бот деплоится на **Railway** с NIXPACKS builder.

- `Procfile` определяет команду запуска
- `requirements.txt` — зафиксированные зависимости
- БД на **Supabase** (PostgreSQL)
