# Воронка продаж (Funnel) — полная документация

## Что это?

Автоматическая 6-дневная воронка продаж, которая запускается после того, как пользователь получил персональный план питания (КБЖУ). Цель — продать 20-минутную тренировку за 990 руб.

---

## Как запускается воронка

### Триггер

Когда AI-агент собрал все данные пользователя и план питания отправлен, вызывается:

```python
# src/handlers/message.py
await set_food_received(chat_id)
```

Эта функция (`src/db/queries.py`) устанавливает:

```sql
UPDATE users_nutrition
SET get_food = TRUE, funnel_stage = 0, last_funnel_msg_at = NOW()
WHERE chat_id = $1
```

С этого момента пользователь — кандидат для воронки.

### Планировщик

**Файл:** `src/funnel/scheduler.py`

APScheduler запускает рассылку **ежедневно в 23:00 UTC**:

```python
scheduler.add_job(_job, "cron", hour=23, minute=0, id="daily_funnel")
```

Настраивается в `src/main.py` при старте бота:

```python
scheduler = setup_scheduler(bot)
scheduler.start()
```

## Таймлайн для пользователя

Допустим, пользователь получил план питания **в 10:00 UTC**:

| Время | Событие | `funnel_stage` |
|-------|---------|:--------------:|
| 10:00 | Получил план питания | 0 |
| 23:00 (тот же день) | Сообщение дня 0 (бесплатная тренировка) | 0 → 1 |
| 23:00 (+1 день) | Сообщение дня 1 (предложение купить) | 1 → 2 |
| 23:00 (+2 дня) | Сообщение дня 2 (отзывы) | 2 → 3 |
| 23:00 (+3 дня) | Сообщение дня 3 (проблемные зоны) | 3 → 4 |
| 23:00 (+4 дня) | Сообщение дня 4 (обоснование цены) | 4 → 5 |
| 23:00 (+5 дней) | Сообщение дня 5 (дедлайн) | 5 → 6 |
| далее | Воронка завершена, сообщений больше нет | 6 |

Первое сообщение приходит в **ближайшие 23:00 UTC** — задержка от 0 до ~24 часов.

## Стадии воронки (RU)

### Стадия 0 — «Разбуди тело» (бесплатная тренировка)

**Текст:** Приветствие + объяснение, почему «спящее тело» мешает результатам. Предложение 7-минутной зарядки без прыжков и нагрузки.

**Кнопка:** `"Разбуди тело"` → callback `video_workout`

**При нажатии:**
1. Бот отправляет URL-кнопку со ссылкой на видео в Google Drive
2. Через **5 минут** автоматически приходит follow-up (`VIDEO_WORKOUT_RESPONSE`) с кнопкой `"Купить со скидкой"` → callback `buy_now`

```
i18n: FUNNEL_DAY_0, FUNNEL_DAY_0_BUTTON
```

### Стадия 1 — Предложение купить

**Текст:** «Тело включилось» → предложение 20-мин тренировки для проблемных зон. Цена: ~~1900₽~~ → **990₽**.

**Кнопка:** `"Купить со скидкой"` → callback `buy_now`

```
i18n: FUNNEL_DAY_1, FUNNEL_DAY_1_BUTTON
```

### Стадия 2 — Социальное доказательство (отзывы)

**Текст:** Типичные результаты: «ушли отёки», «живот стал меньше», «целлюлит мягче», «пропала тяжесть».

**Кнопки:**
- `"Да, хочу результат!"` → callback `buy_now`
- `"Тренировка подойдёт мне?"` → callback `check_suitability` (отправляет видео)

```
i18n: FUNNEL_DAY_2, FUNNEL_DAY_2_BUTTONS
```

### Стадия 3 — Проблема → решение

**Текст:** Список проблем (отёки, ушки, целлюлит, сутулость) → причина (лимфа, глубокие мышцы, осанка) → решение (20-минутная тренировка).

**Кнопки:**
- `"Да!"` → callback `buy_now`
- `"Что в программе?"` → callback `show_info` (отправляет видео)

```
i18n: FUNNEL_DAY_3, FUNNEL_DAY_3_BUTTONS
```

### Стадия 4 — Обоснование цены

**Текст:** 990₽ — это меньше, чем ужин, тушь, кофейня, доставка. Эффект как 2-3 персональные тренировки. 20 минут дома, без инвентаря.

**Кнопки:**
- `"Хочу скидку"` → callback `buy_now`
- `"Мне не подходит"` → callback `none` (мягкий отказ + направление в канал)

```
i18n: FUNNEL_DAY_4, FUNNEL_DAY_4_BUTTONS
```

### Стадия 5 — Мягкий дедлайн (последний день)

**Текст:** «Сегодня последний день за 990₽, дальше цена вернётся». Список результатов: лёгкие ноги, минус в животе, осанка, ягодицы.

**Кнопки:**
- `"Да, беру!"` → callback `buy_now`
- `"Напомни позже"` → callback `remind_later` (направление в канал)

```
i18n: FUNNEL_DAY_5, FUNNEL_DAY_5_BUTTONS
```

## Callback-обработчики

**Файл:** `src/handlers/callbacks.py`

| Callback | Стадия | Действие |
|----------|--------|----------|
| `video_workout` | 0 | Отправляет URL-кнопку на видео + через 5 мин follow-up с `buy_now` |
| `buy_now` | 1-5 | `mark_as_buyer()` → отправляет ссылку на оплату |
| `check_suitability` | 2 | Скачивает и отправляет видео «подойдёт ли мне» с Google Drive |
| `show_info` | 3 | Скачивает и отправляет видео «что в программе» с Google Drive |
| `none` | 4 | Мягкий отказ: «питание + тренировки = супер-комбо» + ссылка на канал |
| `remind_later` | 5 | «Буду рад видеть тебя в канале» + ссылка на канал |

### Оплата

- **RU пользователи:** ссылка Tribute (`settings.tribute_link`)
- **EN/AR пользователи:** ссылка Ziina (`settings.ziina_link`), fallback на Tribute

При нажатии `buy_now`:
1. Вызывается `mark_as_buyer(user_id)` → `is_buyer = TRUE`
2. Пользователь **исключается** из всех будущих рассылок воронки
3. Отправляется URL-кнопка на страницу оплаты

## Механика рассылки

**Файл:** `src/funnel/sender.py`

### Алгоритм `send_funnel_messages(bot)`

1. Получает список целей: `get_funnel_targets()`
2. Для каждого пользователя:
   - Определяет `stage` и `language`
   - Вызывает `get_funnel_message(stage, language)` → текст + кнопки
   - Отправляет сообщение через Telegram API
   - **После успешной отправки** увеличивает `funnel_stage += 1`
3. Rate limiting: пауза 1 сек после каждых 25 сообщений

### Выборка целевых пользователей

```sql
-- src/db/queries.py → get_funnel_targets()
SELECT chat_id, funnel_stage, language
FROM users_nutrition
WHERE (is_buyer IS FALSE OR is_buyer IS NULL)
  AND get_food = TRUE
  AND funnel_stage >= 0
  AND funnel_stage < 5
```

**Критерии:**
- `is_buyer = FALSE` — не покупатель
- `get_food = TRUE` — получил план питания
- `funnel_stage` от 0 до 4 (стадия 5 отправляется при stage=4→5, потом 5→6 и пользователь выходит)

### Rate limiting

```python
_BATCH_SIZE = 25   # сообщений в пачке
_BATCH_DELAY = 1.0  # секунд между пачками
```

Telegram допускает ~30 сообщений/сек в разные чаты, так что это консервативно.

## Фабрика сообщений

**Файл:** `src/funnel/messages.py`

```python
@dataclass
class FunnelMessage:
    text: str
    buttons: list[tuple[str, str]]  # [(label, callback_data), ...]
    has_url_button: bool = False
    url: str = ""
```

Функция `get_funnel_message(stage, language)` возвращает `FunnelMessage` для стадии 0-5 или `None` если стадия вне диапазона.

Тексты и кнопки берутся из i18n модулей (`src/i18n/ru.py`, `en.py`, `ar.py`) через `get_strings(language)`.

## Важные нюансы

### 1. Инкремент стадии после отправки

Стадия увеличивается **сразу после** успешной отправки сообщения в `sender.py`. Это значит: когда пользователь нажимает кнопку из сообщения стадии N, в базе он уже на стадии **N+1**.

### 2. Защита от гонки (`advance_funnel_if_at_stage`)

Для 5-минутного follow-up после `video_workout` используется условный инкремент:

```python
# Увеличить stage только если пользователь ещё на стадии 1
await advance_funnel_if_at_stage(user_id, expected_stage=1)
```

Это предотвращает двойной инкремент, если планировщик уже успел обработать пользователя.

### 3. Стадия 5 — финальная

- Планировщик отправляет стадии 0-4 (условие `< 5` в SQL)
- При отправке стадии 4 stage становится 5
- При stage=5 пользователь ещё попадает в выборку... **Нет!** Условие `funnel_stage < 5` — стадия 5 **не отправляется** планировщиком повторно
- Фактически: stage 0→1→2→3→4→5 (отправлено 5 сообщений), при 5→6 сообщение не отправляется

**Исправление:** Стадии 0-5 **включительно** отправляются (`< 5` в SQL = стадии 0,1,2,3,4 отправляются; stage 5 — последняя, при ней stage был 4, после отправки стал 5). Итого 5 сообщений из 6 определённых?

**Верный подсчёт:**
- stage=0: отправляется FUNNEL_DAY_0, stage → 1
- stage=1: отправляется FUNNEL_DAY_1, stage → 2
- stage=2: отправляется FUNNEL_DAY_2, stage → 3
- stage=3: отправляется FUNNEL_DAY_3, stage → 4
- stage=4: отправляется FUNNEL_DAY_4, stage → 5
- stage=5: **НЕ попадает** в `funnel_stage < 5` → FUNNEL_DAY_5 **не отправляется**

> **BUG:** Сообщение стадии 5 (FUNNEL_DAY_5) никогда не отправляется, т.к. SQL-условие `funnel_stage < 5` исключает stage=5. Нужно изменить на `funnel_stage < 6` или `funnel_stage <= 5`.

### 4. Тестовый аккаунт

`chat_id = 379336096` — тестовый аккаунт, для которого `set_food_received()` не вызывается (можно пересчитывать КБЖУ без ограничений).

Скрипт для сброса: `./reset_test_user.sh`

## Поля БД (таблица `users_nutrition`)

| Поле | Тип | Описание |
|------|-----|---------|
| `get_food` | boolean | `TRUE` = получил план питания, кандидат для воронки |
| `funnel_stage` | integer | Текущая стадия (0-6). 6 = воронка завершена |
| `last_funnel_msg_at` | timestamp | Время последнего сообщения воронки |
| `is_buyer` | boolean | `TRUE` = купил тренировку, исключён из воронки |
