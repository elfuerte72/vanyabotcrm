# Telegram Bot VANYA_BOT — Полная документация

## 1. Обзор

AI-нутрициолог на базе Telegram. Собирает данные пользователя через диалог, рассчитывает КБЖУ по формуле Mifflin-St Jeor, генерирует план питания и проводит 6-дневную воронку продаж.

**Стек:** Python 3.11+ / aiogram 3.x / asyncpg / OpenRouter (Gemini 3 Flash) / PostgreSQL (Railway)

---

## 2. Поток сообщений

```
Пользователь отправляет текст/голос
         │
         ▼
[LoggingMiddleware] — логирует входящее событие
         │
         ▼
[UserDataMiddleware] — загружает User из БД
         │
         ▼
   get_food == TRUE?
   ├─ ДА → отправляет ALREADY_CALCULATED на языке юзера → СТОП
   │
   └─ НЕТ → detect_language() → AGENT MAIN (OpenRouter LLM)
                │
                ▼
         parse_agent_output()
                │
         route_type?
         ├─ "conversation" → отправляет текст ответа юзеру
         │
         └─ "generate" (is_finished=true)
              ├─ calculate_macros() (Mifflin-St Jeor)
              ├─ save_user_data() → БД
              ├─ set_food_received() → funnel_stage=0
              ├─ AGENT FOOD → JSON с планом питания
              ├─ validate_meal_plan() → ±10% калорий
              ├─ format_meal_plan_html()
              └─ отправка HTML меню юзеру
```

## 3. Расчёт КБЖУ (Mifflin-St Jeor)

### Формула BMR
- **Мужчины:** `10 × вес(кг) + 6.25 × рост(см) - 5 × возраст + 5`
- **Женщины:** `10 × вес(кг) + 6.25 × рост(см) - 5 × возраст - 161`

### Коэффициенты активности (TDEE = BMR × коэффициент)

| Уровень | Коэффициент | Описание |
|---------|-------------|----------|
| sedentary | 1.2 | Сидячий образ жизни |
| light | 1.375 | Лёгкая активность |
| moderate | 1.55 | Умеренная активность |
| high | 1.725 | Высокая активность |
| extreme | 1.9 | Экстремальная активность |

### Корректировка по цели

| Цель | Формула |
|------|---------|
| weight_loss | TDEE × 0.85 (дефицит 15%) |
| maintenance | TDEE (без изменений) |
| muscle_gain | TDEE × 1.10 (профицит 10%) |

### Распределение макронутриентов

| Нутриент | Формула |
|----------|---------|
| Жиры | 1.0 г/кг массы тела |
| Белки | 1.5 г/кг (похудение/набор), 1.4 г/кг (поддержание) |
| Углеводы | (калории - белки×4 - жиры×9) / 4, минимум 0 |

### Примеры расчётов

| Профиль | Калории | Белки | Жиры | Углеводы |
|---------|---------|-------|------|----------|
| М, 80кг, 180см, 30 лет, sedentary, weight_loss | 1816 | 120 | 80 | 199 |
| Ж, 60кг, 165см, 25 лет, moderate, maintenance | 2085 | 84 | 60 | 342 |
| М, 100кг, 190см, 40 лет, high, muscle_gain | 3781 | 150 | 100 | 618 |
| Ж, 65кг, 170см, 28 лет, moderate, weight_loss | 1860 | 98 | 65 | 221 |

## 4. Воронка продаж (6 дней)

Запускается после получения плана питания (`set_food_received` → `funnel_stage=0`).
Ежедневно в 23:00 UTC крон отправляет сообщения пользователям со stage 0-4.
После каждой отправки `funnel_stage` увеличивается на 1.

### Что протестировано

- **КБЖУ калькулятор:** Формула Mifflin-St Jeor для всех комбинаций пол/активность/цель, граничные значения (40-200 кг, 16-80 лет), идентификаторы пола (male/m/м)
- **База данных:** UPSERT, get/save/update, funnel targets (исключение buyers, фильтр по stage), chat history (порядок, JSONB формат)
- **Callback-кнопки:** Все 7 хендлеров на 3 языках, проверка текстов, кнопок, URL оплаты, обработка ошибок
- **Воронка продаж:** 6 стадий × 3 языка (18 проверок содержимого), sender logic, полный 6-дневный цикл, buyer exclusion
- **Message flow:** Guard (get_food), route conversation/generate, определение языка, расчёт КБЖУ, интеграция с agent_food
- **E2E:** Полный путь пользователя от создания до покупки через реальную БД
